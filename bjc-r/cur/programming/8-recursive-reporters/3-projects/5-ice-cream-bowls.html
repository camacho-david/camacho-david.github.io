Por ahora, todas las bolas deben ser diferentes entre sí.

Una tienda tiene ocho sabores de helado. Puede pedir un tazón que tenga 1, 2 o 3 cucharadas diferentes. No importa el orden de las bolas: un bol con vainilla y chocolate es lo mismo que un bol con chocolate y vainilla.

  1. ¿Cuántos tazones de 2 cucharadas hay? Encuentre una forma sistemática de enumerarlos.
  2. Busque una manera de describir el problema usando la recursividad.
  3. Es posible que desee utilizar el bloque append que se ve en Unidad 8 Práctica de laboratorio 2 Página 3: Ordenacion de partición.
    Cree un bloque que informe una lista de listas, todos los tazones de 2 cucharadas dados una lista de entrada de longitud variable de sabores disponibles.
El bloque que construiste para tazones de 2 cucharadas puede ayudarte a construir un bloque para tazones de 3 cucharadas.
  1. Cree un bloque que enumere todos los tazones de 3 cucharadas, dada una lista de los sabores disponibles. Recuerde, el orden de las cucharadas no importa, así que piense en cómo podría evitar enumerar el mismo tazón más de una vez.
  2. Ese tazón tiene chocolate o no...
    Generalice para crear un bloque que enumere todos los tazones de n cucharadas, con una entrada adicional que proporcione el número de cucharadas. ¡Esto definitivamente necesitará recursividad!
  1. Elimine la restricción de que cada cucharada debe ser de un sabor diferente: ahora se permite chocolate, chocolate y chocolate, y muchos más.
